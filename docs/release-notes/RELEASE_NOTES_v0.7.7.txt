Release Notes - v0.7.7

Date: 2026-04-21

Scope

- repository/app version: 0.7.7
- release type: focused Local Whisper runtime-planning follow-up
- validation surface: Rig1

What changed

- Tuned Local Whisper runtime planning for short `large` GPU runs on Rig1.
- Added a conservative short-run heuristic guardrail for `large` GPU planning
  so still-progressing CUDA transcription is less likely to be killed too early
  before calibration data exists.
- Widened runtime calibration recommendation for short `large` GPU runs and
  raised the calibration sample floor for that case to `45s`.
- Mirrored the same short-run guardrail/calibration behavior in the PowerShell
  fallback planner path used by the wrappers.
- Added targeted planner tests for the short `large` GPU case while preserving
  the existing short `medium` GPU behavior.

Why this matters

- The preserved Rig1 Brooklands `local-large-gpu` comparison did not fail
  because of first-output/bootstrap deadlines, stall-watchdog expiry,
  GPU-to-CPU fallback, or CUDA/device-selection failure.
- It failed because the adaptive runtime budget was too optimistic for a
  `5m 22s` source on this machine, so the run was terminated while CUDA
  transcription was still reporting progress.
- This patch keeps the fix narrow: it adjusts runtime planning for the
  affected `large` GPU case without broadening into new benchmark suites,
  Video parity work, or unrelated Whisper behavior changes.
