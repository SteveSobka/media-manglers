Audio Mangler
=============

Who it is for
-------------

Audio Mangler is for audio-first review. Use it when the spoken content matters
more than pictures.

Good fits:

- podcasts
- interviews
- lectures
- speeches
- voice notes
- multilingual audio that needs both the original transcript and translated
  versions

What it makes
-------------

Each completed package is built around the original spoken source and usually
includes:

- audio\review_audio.mp3
- transcript\transcript_original.srt
- transcript\transcript_original.json
- transcript\transcript_original.txt
- translations\<lang>\transcript.srt / .json / .txt when requested
- comments\comments.txt / .json when requested and available from a YouTube
  source
- segment_index.csv
- README_FOR_CODEX.txt
- script_run.log

Optional extras:

- raw\ original source audio copy
- chatgpt_review_package.zip

How translation works
---------------------

Audio Mangler always creates the original-language transcript first.

If you request translation, it then creates translated transcript files from
that original transcript and source audio. The app is designed to prefer the
original spoken source, not weak platform auto-translation.

Mode choices:

- Local mode: local transcription plus local translation
- AI Private: OpenAI transcription plus OpenAI translation
- AI Public: local transcription plus OpenAI translation on the Public/shared
  project

That difference matters. AI Public does not behave the same way as AI Private,
and AI Public does not imply that source audio is uploaded to OpenAI for
transcription.

Local translation paths:

- to English: can use Whisper locally
- to other target languages: can use Argos Translate when the language packages
  are installed

Local mode now defaults to Whisper large. That is intentional: local accuracy
and nuance are prioritized over speed, so Local runs will be slower and heavier
than smaller Whisper models. Advanced users can still choose a different
supported local Whisper model with -WhisperModel.

If local translation support is missing, the app explains what is missing, what
it unlocks, and gives you install commands plus an install/skip/cancel choice.
It does not auto-install dependencies behind your back.

OpenAI API key setup
--------------------

Create the key in your OpenAI Platform account:

  https://platform.openai.com/api-keys

Current OpenAI paths in code:

- AI translation uses POST /v1/chat/completions
- AI Private transcription uses POST /v1/audio/transcriptions

Recommended setup for normal local use:

- Create the key in your OpenAI Platform account.
- Choose Owned by you.
- Put Media Manglers in a dedicated project.
- Choose Restricted.
- In the current OpenAI Platform UI, turn on Request for Chat Completions
  (/v1/chat/completions).
- For AI Private, also turn on Request permission for audio transcription
  access because Private AI mode sends source audio to OpenAI for
  transcription.
- Leave unrelated permissions like Images, Embeddings, Files, Fine-tuning,
  Vector Stores, Assistants, Batches, and similar extras off or set to None
  unless you actually use them.
- Read Only is not enough because Audio Mangler sends POST
  /v1/chat/completions requests for translation.
- Service accounts are mainly for shared automation, servers, CI, or other
  non-personal bot identities, not normal desktop use.
- ChatGPT subscriptions and OpenAI API billing are separate.
- OpenAI translation needs API billing/credits on the selected OpenAI Platform
  project.
- OpenAI API usage may incur charges.
- Private is the default and safest key profile.
- If you want a Private/Public split, create two separate OpenAI Platform
  projects and one key per project.
- Use OPENAI_API_KEY_PRIVATE for the default Private path.
- Use OPENAI_API_KEY_PUBLIC only when you explicitly run -ProcessingMode AI
  -OpenAiProject Public.
- OPENAI_API_KEY still works as a legacy Private fallback for older setups.
- The script cannot infer whether a pasted key is Private or Public.
- Use Private for confidential, sensitive, internal, client, or proprietary
  media. If you are unsure, use Private.
- Use Public only for media you are allowed to share with OpenAI.
- Public/shared complimentary tokens only apply when the Public project is
  configured for shared traffic and the request uses an eligible model.
- If a request would cross the remaining complimentary daily quota, OpenAI
  bills the whole request normally.

Ways to provide the key on Windows:

- Paste it at the prompt for the current run only.
- Current PowerShell session:

  $env:OPENAI_API_KEY_PRIVATE="sk-..."

- Public project session:

  $env:OPENAI_API_KEY_PUBLIC="sk-..."
  powershell -NoProfile -ExecutionPolicy Bypass -File '.\Audio Mangler.ps1' -TranslateTo en -ProcessingMode AI -OpenAiProject Public

- Legacy fallback:

  $env:OPENAI_API_KEY="sk-..."

- Persistent Windows user variable:

  [System.Environment]::SetEnvironmentVariable("OPENAI_API_KEY_PRIVATE","sk-...","User")

After setting the persistent user variable, open a new PowerShell window before
rerunning the app. Do not hardcode the key in the script or commit it to GitHub.

Common commands
---------------

Run the script:

  powershell -NoProfile -ExecutionPolicy Bypass -File '.\Audio Mangler.ps1'

Show the version:

  powershell -NoProfile -ExecutionPolicy Bypass -File '.\Audio Mangler.ps1' -Version

Translate to English with Local mode:

  powershell -NoProfile -ExecutionPolicy Bypass -File '.\Audio Mangler.ps1' -TranslateTo en -ProcessingMode Local

Translate to English and Spanish with the local path:

  powershell -NoProfile -ExecutionPolicy Bypass -File '.\Audio Mangler.ps1' -TranslateTo en,es -ProcessingMode Local

Use AI Private for OpenAI transcription plus OpenAI translation:

  powershell -NoProfile -ExecutionPolicy Bypass -File '.\Audio Mangler.ps1' -TranslateTo en -ProcessingMode AI -OpenAiProject Private

Use AI Public for local transcription plus OpenAI translation on the
Public/shared project:

  powershell -NoProfile -ExecutionPolicy Bypass -File '.\Audio Mangler.ps1' -TranslateTo en -ProcessingMode AI -OpenAiProject Public

Request YouTube comments when available:

  powershell -NoProfile -ExecutionPolicy Bypass -File '.\Audio Mangler.ps1' -IncludeComments

Interactive YouTube runs also ask:

  If comments are available for a YouTube source, save them in the package too? (Y/n):

Press Enter to accept comments by default. Type N or No to skip them.

Keep temporary processing files for troubleshooting:

  powershell -NoProfile -ExecutionPolicy Bypass -File '.\Audio Mangler.ps1' -KeepTempFiles

Validation
----------

Smoke test:

  powershell -NoProfile -ExecutionPolicy Bypass -File .\AREA51\Run-AudioSmokeTest.ps1

Translation smoke test:

  powershell -NoProfile -ExecutionPolicy Bypass -File .\AREA51\Run-AudioSmokeTest.ps1 -TranslateTo en -ProcessingMode Local

Validator:

  powershell -NoProfile -ExecutionPolicy Bypass -File .\AREA51\Validate-AudioManglerPackage.ps1

Notes
-----

- Local mode keeps transcription and translation on your machine after its
  dependencies are installed.
- AI Private sends source audio plus transcript/translation content to OpenAI.
- AI Public keeps transcription local and only sends transcript/translation
  content to the Public/shared OpenAI project.
- Temporary helper files and staging folders are cleaned up by default unless
  you use -KeepTempFiles.
